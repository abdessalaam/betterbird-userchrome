# Contributing Guidelines

Thank you for contributing.

## Principles

- Maintain clarity and modularity.
- Prefer additive changes over destructive edits.
- Keep CSS readable and properly commented.
- Avoid unnecessary specificity.
- Do not introduce breaking changes without discussion.

## Pull Requests

1. Fork the repository.
2. Create a feature branch.
3. Make changes.
4. Submit a Pull Request to `main`.

At least one review is required before merge.

## Reporting Issues

Please include:

- Betterbird version
- Operating system
- Screenshot (if UI-related)
- Clear reproduction steps
